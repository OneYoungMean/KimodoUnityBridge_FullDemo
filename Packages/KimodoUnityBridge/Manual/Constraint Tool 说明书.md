# Constraint Tool

## 概述

约束（Constraint）是你对生成动作施加精确控制的方式。

提示词告诉 AI "做一个什么样的动作"，但它无法说清"手要伸到哪个确切位置""这一帧应当摆出什么姿势"。约束正是用来补足这一点的。你在动画轨道上放置一个约束标记（Marker），它告诉生成过程：到这一帧，身体或某个肢体末端必须到达指定的位置、摆出指定的姿势。生成的动作便会经过这个点。

借助约束，你可以让手准确够到物体、让脚踩在固定落点、让角色在某一帧摆出特定造型。它与 Timeline 配合使用，能让动画"听话"很多。

## 约束的种类

约束以 Kimodo 约束 Marker 的形式添加到 Animation Track 上。按你要控制的内容，可分为以下几类。

| 类型 | 控制对象 | 适用场景 |
| --- | --- | --- |
| **FullBody** | 整个身体的姿势（根位置 + 各关节角度） | 需要在某一帧精确摆出目标造型 |
| **Root2D** | 角色在地面（X/Z 平面）的位置，可选朝向 | 走位路线、移动方向、转身控制 |
| **Left / Right Hand** | 左/右手末端的位置与朝向 | 抓取、挥手、指向 |
| **Left / Right Foot** | 左/右脚末端的位置与朝向 | 落脚点、迈步目标、防止脚滑 |
| **自定义 End-Effector** | 可同时指定多个末端目标 | 一只手加一只脚等混合控制 |

## 两种工作方式

每个约束 Marker 都有一个 **useOverride** 开关，它决定约束的数值从哪里来。理解这个开关，是用好约束的关键。

**自动采样（useOverride 关闭）**

这是默认状态。Marker 会自动读取它所在那一帧、原始动作中的姿势，并把该姿势作为约束。此时 Inspector 中显示的数值是只读的采样结果。

适用情况：你已有一段动作，只想把其中某一帧的姿势"钉住"，让重新生成的动作在这个点保持不变。

**手动覆盖（useOverride 开启）**

开启后，约束数值交由你自己控制。你可以手动修改根位置、关节角度等，让生成动作遵循你设定的姿势，而非照搬原动作。

## Marker 的 Inspector 面板

选中一个约束 Marker，Inspector 中会显示以下内容：

- **说明文字**：顶部的提示说明这个约束的用途和适用场景。
- **useOverride**：前述的自动/手动开关。
- **Refresh Cache**：强制重新采样姿势并重建预览。修改过原动作、或预览看起来不对时点它刷新。
- **Edit**：打开专门的姿势编辑窗口（见下节）。点击它会自动开启 useOverride。
- **Marker Time (seconds)**：约束在时间轴上的位置（秒），下方会显示换算后的采样时间。修改它可把约束移到其他帧。
- **姿势数据字段**：随约束类型不同，显示根位置、关节角度、朝向、关节名等。useOverride 开启时可编辑，关闭时为只读的采样结果。

## 用编辑窗口调整姿势

点击 **Edit** 会打开 **Constraint Override Edit** 窗口，这是手动调整姿势最直接的地方。

它的特点是实时写入：窗口打开期间，你在场景视图中拖动约束姿势的关节，数据会立即写回 Marker，预览随之更新。你无需反复保存，调到满意直接关闭窗口即可。窗口中也显示姿势数值，需要精确输入时可在此修改。

> 编辑窗口要求 useOverride 处于开启状态。如果在窗口打开时关闭了 useOverride，窗口会提示它已失效。

## 典型流程：从已有动作中采集姿势

约束有一个很实用的用法——先生成、再从中挑选姿势、再用它约束：

1. 先用 Timeline 生成一段 Kimodo 动作，观察其中是否有你喜欢的姿势。
2. 在喜欢的那一帧放置一个约束 Marker（例如 FullBody）。
3. 保持 useOverride 关闭，它会自动采样那一帧的姿势作为约束。
4. 这样你便"采集"到了一个理想姿势，可用它去约束后续的生成。

若想在采样基础上微调，开启 useOverride，进入编辑窗口拖动调整即可。

## 让约束生效

在 Timeline 片段的 Inspector 中勾选 **Show Constraint**，即可在场景视图中看到该片段范围内所有约束姿势的预览，确认它们的摆放是否正确。

约束设置好之后，正常点击片段的 **Generate & Bake** 重新生成，生成过程会把这些约束纳入考虑，使动作经过你指定的姿势和位置。

## 注意事项

- 约束 Marker 必须放在它所属片段的时间范围之内，才会被采样和生效；放到片段范围外不会计入。
- 编辑保存偶尔有概率未写入成功。若发现修改未生效，点击 **Refresh Cache** 或重新打开编辑窗口确认。
- 约束放得越多、彼此越矛盾，生成越难同时满足，结果可能变形。建议从少量关键约束开始，不足再补。
- 自定义 End-Effector 的关节名可填 LeftHand / RightHand / LeftFoot / RightFoot / Hips 等，用于同时控制多个目标。
