# Animator Tool

## 概述

Animator Tool 把 Kimodo 的动画生成能力直接带到了 Animator Controller 里。

Animator Controller 是一个状态机，它管理着角色在不同时刻播放哪个动画片段，以及片段之间如何过渡。Animator Tool 让你针对状态机中的某个状态（State）或某条过渡（Transition），用一句提示词生成新的动作，并把它直接接入状态机——你不再需要先在 Timeline 上把动画烤好，再手动搬运过来。

它适合两类工作：替换某个状态里你不满意的动作，或者在两个状态之间插入一段自然的衔接动画。

## 打开工具

在菜单栏选择 **Kimodo → Kimodo Animator Tool**，打开工具窗口。

窗口分为两部分：左侧是预览区，实时播放当前选中对象的动作；右侧是参数区，负责调参和生成。

## 选择操作对象

这个工具跟随你的选择工作。在 Animator 窗口中选中一个状态，或选中一条过渡连线，工具会自动识别，左侧预览区随即播放对应的动作。

- 选中**状态**：生成的动画将替换这个状态原有的片段。
- 选中**过渡**：生成的动画将作为一段新的衔接动作，插入两个状态之间。

预览区上方提供两种查看方式：

- **Show Original**：查看选中对象原本的动作。
- **Show Generated**：查看生成后的动作，便于前后对比。

旁边的 **Pre(s)** 与 **Post(s)** 用于在过渡预览时前后多留几秒余量，让你看清衔接是否顺畅。若想固定当前选择不被误触打乱，可使用 **Reselect** 重新抓取。

> 目前暂不支持 BlendTree（混合树）类型的选择，选中它时工具会提示你更换。

## 生成设置

右侧 **Generate** 区域是核心。各项含义如下：

| 设置 | 说明 |
| --- | --- |
| **Backend** | 生成后端，保持默认的 Kimodo Bridge 即可（本地运行）。 |
| **Bridge Model** | 用于桥接生成的 Kimodo 模型包，使用已安装的默认项即可。 |
| **VRAM Mode** | Low 占用约 4G 显存，High 约 16G、质量更稳。按显卡情况选择。 |
| **Prompt** | 自然语言动作描述。选中对象时会自动填入一句参考词，可直接修改。 |
| **Auto / Duration (s)** | 勾选 Auto 沿用原动作时长；取消勾选可拖动滑条自定义秒数。 |
| **Diffusion Steps** | 生成采样步数，越高越精细也越慢。默认值适用于大多数情况。 |
| **InOut Constraint** | 首尾约束模式，见下表。 |
| **Is Loop** | 仅在 Inside 模式下可用。让动作首尾相接，适合循环动作。 |
| **Random / Seed** | 勾选 Random 每次结果不同；取消勾选填固定 Seed 复现同一结果。 |

InOut Constraint 的三种模式：

| 模式 | 含义 |
| --- | --- |
| **None** | 不施加首尾约束，动作自由发挥。 |
| **Inside** | 使用这段动作自身的起始/结束姿势作约束，适合循环。 |
| **Outside** | 使用相邻动作的边界姿势作约束，让衔接更自然。 |

## 完整流程

1. 在 Animator 窗口中选中一个状态，或一条过渡。
2. 在左侧预览区确认这是你要修改的动作。
3. 在右侧填写提示词，调整时长、约束等参数。
4. 点击 **Generate & Bake**，等待生成完成。CUDA 大约几秒，纯 CPU 会慢不少；生成过程中可点 **Cancel** 中止。
5. 生成完成后切到 **Show Generated** 查看效果。不满意可改写提示词或勾选 Random 重新生成，**Reset** 可清除当前结果重来。
6. 满意后点击底部的 **Apply**，动画正式接入状态机：
   - 若选中的是状态，新动作替换它原有的片段。
   - 若选中的是过渡，工具会自动新建一个状态插入中间，命名形如 `起点_终点_KimodoInsert`，并接好过渡关系。

## 注意事项

- 第一次生成会自动下载模型并配置环境，需要等待一段时间，属于正常现象。偶尔因网络波动失败，重新生成通常即可解决。
- 生成基于你选中对象当前的动作进行，因此选对对象很关键。
- **Apply** 会实际改动你的 Animator Controller，建议操作前留好备份，或先确认预览满意再执行。
