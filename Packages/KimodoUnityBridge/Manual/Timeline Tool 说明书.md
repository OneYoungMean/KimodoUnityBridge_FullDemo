# Timeline Tool

## 概述

Kimodo 的 Timeline 工具建立在动画片段（Animation Clip）的概念之上。一个动画片段记录了角色随时间发生的位置、旋转和姿态变化——你可以把它想象成一段动作的录像。

和传统流程不同的是，这段"录像"不需要你提前准备好。你在 Timeline 上放一个 Kimodo 片段，用一句自然语言描述你想要的动作，工具就会在本地把对应的人形动画生成出来，烤进这个片段，铺到时间轴上。

时间轴负责安排这些片段的先后、长短和衔接。当你把多个片段排在一起，再配合首尾约束，就能组织出一段完整连贯的表演——长动画、循环动画、过渡动画，都从这里来。

## 工作原理

一次完整的生成涉及三个角色：你的角色对象、绑定它的动画轨道，以及轨道上的 Kimodo 片段。

下面的流程说明了它们如何配合：

1. 场景中的角色对象提供骨骼和 Avatar，决定动画最终落在谁身上。
2. Timeline 上的一条 Animation Track 绑定到这个角色。
3. 轨道上的 Kimodo 片段接收你的提示词，在本地生成动作。
4. 生成的动作自动重定向到角色，烤成一个可直接播放的 Unity 动画片段。

工具会自动判断角色类型并适配：Humanoid 角色拿到肌肉动画，Generic 角色拿到骨骼动画。重定向的过程由工具接管，你不需要手动处理。

## 准备工作

开始之前，你的场景里需要具备：

- 一个带 Animator 的人形角色对象。
- 一个 Timeline，其中包含一条绑定到该角色的 Animation Track。
- 这条轨道上放置了一个 Kimodo 片段。

如果你从 Sample 导入了示例场景，以上内容都已搭好，打开即可使用。

## 片段的设置面板

在 Timeline 窗口中选中一个 Kimodo 片段，Inspector 会显示它的全部设置。面板自上而下分为几组。

### Generate Motion

这是生成动作的主区域。

| 设置 | 说明 |
| --- | --- |
| **Backend** | 选择生成后端。默认为本地运行的 Kimodo Bridge；选 ComfyUI 时需要额外填写 IP 和端口。 |
| **Bridge Model** | 用于桥接生成的 Kimodo 模型包，使用已安装的默认项即可。 |
| **VRAM Mode** | Low 使用量化过的文本编码器（约 4G）；High 使用完整编码器（约 16G），质量更稳。面板会估算当前模式所需显存。 |
| **Prompt** | 发送给生成后端的自然语言动作描述。描述越具体，结果越贴近预期。 |
| **Duration (s)** | 生成片段的目标时长。改动它会同步更新时间轴上片段的长度。 |
| **Diffusion Steps** | 生成的采样步数。步数越高耗时越长，细节可能更好。默认值适用于大多数情况。 |
| **Random / Seed** | 勾选 Random 时每次生成使用随机种子；取消勾选时可填固定 Seed 复现同一结果。 |

### InOut Constraint

首尾约束决定生成的动作要不要和上下文对齐，是组织长动画和过渡的关键。

| 模式 | 含义 |
| --- | --- |
| **None** | 不施加首尾约束，动作自由发挥。 |
| **Inside** | 使用这个片段自身的起始/结束姿势作为约束，适合循环动作。 |
| **Outside** | 使用相邻片段的边界姿势作为约束，让这一段与前后自然衔接。 |

勾选 **Show Constraint**，可以在场景视图中直接看到约束姿势的预览，确认它们的位置是否正确。

### Animation Clip

这里是烤好之后的 Unity 动画片段本体，以及两个播放开关：

- **Foot IK**：播放时启用脚部 IK，减少脚滑和穿地。
- **Loop**：当时间轴播放超出片段长度时循环这一段。

### Animation Bake

- **Auto Retarget On Binding**：默认开启，生成的动作会在绑定时自动重定向到轨道绑定的角色。关闭后需要你指定一个 Humanoid Avatar。
- **Advanced**：包含约束归一化和曲线关键帧精简的选项。生成曲线的关键帧过密可能导致卡顿，可在此调整精简误差（Position / Rotation / Float Error）把冗余帧压掉。一般保持默认，遇到卡顿再调整。

## 生成一段动作

最简单的流程如下：

1. 打开 Timeline，选中一条绑定了角色的 Animation Track 上的 Kimodo 片段。
2. 在 Inspector 中填写提示词，调整时长、显存模式等参数。
3. 若需要循环或过渡，设置好 InOut Constraint。
4. 点击 **Generate & Bake**，等待生成完成。CUDA 大约几秒，CPU 会慢一些；生成过程中可点 **Cancel** 中止。
5. 运行 Timeline 查看效果。
6. 不满意时改写提示词，或勾选 Random 再生成一次。**Reset** 可清除本次生成的记录重来。

## 组合多个片段

单个片段有长度上限。要做更长、更连贯的表演，思路是把多个片段排在一起，用首尾约束让它们相接。

- **接力做长动画**：把后一段的 InOut Constraint 设为 Outside，它会对齐前一段的结尾姿势，两段之间不会出现跳变。逐段接力，长动画就成型了。
- **做循环动画**：使用 Inside 模式，让动作首尾相接、转一圈回到起点。
- **做过渡动画**：在两段已有动作之间放一段新的，用 Outside 让它同时照顾前后边界，自然地把两边缝合。

## 注意事项

- 第一次生成会自动下载模型并配置环境，需要等待一段时间，属于正常现象。偶尔因网络波动失败，重新生成通常即可解决。
- 改动 Duration 会同步改变时间轴片段的长度，这是有意为之，便于两边对齐。
- 想精确控制某些帧的姿势，请参阅 **Constraint Tool** 一节，了解如何在片段上放置约束 Marker。
